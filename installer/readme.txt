Analytics Reporting UI
Version 1.0

A modern interface for viewing report data generated by Webtrends Analytics.  Information is retrieved using the Data Extraction API.

----------

Requirements:

Windows Server 2016 or higher
Internet Information Services 10

----------

Prerequisites:

You must already have existing Webtrends Analytics and Data Extraction (DX) API servers.

----------

Installation:

To deploy the Analytics Reporting UI automatically:
 1. Run the installer (wtr_setup.exe).
 2. Select 'Yes' when asked if IIS should be installed.
 3. Provide the DX API endpoint.
     -This is where report data will be retreived.
 4. Provide the URL of the Analytics 9 UI.
     -This is used for the Analytics 9 link in the account menu.
 5. Provide a port number for the Analytics Reporting UI IIS site.

To deploy the Analytics Reporting UI manually:
 1. Install the Web Server (IIS) with the default features.
 2. Install URL Rewrite 2.1.
     -The installer is found in the zip file or can be downloaded here:
	https://www.iis.net/downloads/microsoft/url-rewrite
 3. Add a new IIS Application Pool with default options.
 4. Add a new IIS site and select the newly created Application Pool.
 5. Copy all contents of the \app folder in the zip to the physical path of the new site.
 6. Create a config.js file from \app\config.js.example, replacing the DX endpoint and Analytics 9 URL with correct values.

----------

Cross Origin Resource Sharing (CORS):

The Webtrends Data Extraction API server may need to be configured to accept cross domain requests.

On the DX API server:

 1. Install the IIS CORS module.
     -https://www.iis.net/downloads/microsoft/iis-cors-module 

 2. Edit the root web.config file.
     -\Webtrends\Data Extraction Website\web.config

 3. Add the CORS configuration in the <system.webServer> section.
     -https://learn.microsoft.com/en-us/iis/extensions/cors-module/cors-module-configuration-reference 
     -For example:
	<cors enabled="true">
    	  <add origin="*" />
	</cors>

 4. Add a custom header for authorization in the <system.webServer> section.
	<httpProtocol>
    	  <customHeaders>
            <add name="Access-Control-Allow-Headers" value="authorization" />
    	  </customHeaders>
	</httpProtocol>
